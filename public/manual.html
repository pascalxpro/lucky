1. 系統概述

Riiqi Lucky 是一套「投票 + 即時抽獎」互動平台，適用於票選活動、節慶抽獎、品牌推廣等場景。

核心流程：參與者瀏覽作品 → 投票 → 觸發抽獎遊戲 → 中獎領獎

系統架構

項目 技術
框架 Next.js 16 (App Router)
資料庫 SQLite（輕量、免安裝）
部署 Docker 容器化

前台功能

功能 說明
🖼️ Banner 輪播 自動 5 秒切換 + 手動滑動
🎨 作品展示 卡片式展示 + 圖片輪播 + 點擊放大
🗳️ 投票 每人每作品一票，設備指紋辨識
🎡 抽獎遊戲 三種模式：轉盤 / 刮刮樂 / 拉霸
🏆 中獎領獎 填寫姓名、手機、地址（可個別關閉）
🎵 音效 內建合成音效，可靜音

4. 活動管理

進入「活動管理」頁面
點擊「新增活動」填寫名稱與說明
選擇遊戲模式（轉盤/刮刮樂/拉霸）
點擊「啟用」讓活動上線（同時僅一個活動可啟用）
啟用新活動時，系統會自動停用其他活動。

5. 作品管理

進入「作品管理」頁面
點擊「新增作品」填寫名稱、描述
上傳圖片（支援多張輪播，建議尺寸 800×600 px (4:3)）
拖曳調整顯示排序
每件作品支援多張圖片，前台會自動以輪播方式展示。

6. 獎品與機率設定

抽獎採用權重式隨機，不是百分比。系統會自動計算各獎品的實際中獎機率。

設定步驟

進入「獎品管理」頁面
新增獎品：填寫名稱、上傳圖片（建議 400×400 px (1:1)）
設定庫存總量與機率權重
設定是否為「安慰獎」（不扣庫存、不算中獎）
設定「中獎後是否需填寫個資」（每個獎品可獨立設定）

機率權重範例

獎品 權重 實際機率 庫存
iPhone 16 Pro 0.5 0.5% 1
AirPods Pro 2 2% 3
星巴克 $500 5 5% 10
精美禮物 15 15% 50
折價券 27.5 27.5% 200
感謝參與 安慰獎 50 50% 無限
權重加總為 100 時，數字即等於百分比。庫存歸零的獎品會自動從抽獎池排除。

10. 防灌票機制

系統提供三層防護，有效阻止灌票行為：

層級 機制 效果
① 設備指紋 基於 Canvas、WebGL、螢幕、CPU 等硬體特徵產生確定性指紋 同瀏覽器 + 同設備 = 同一人（含無痕模式）
② 每人票數上限 依設備指紋計算已投票數 限制總投票數量
③ IP 限制 同一 IP 投票數上限（後台可設定） 防止換瀏覽器/無痕灌票
建議 IP 上限設為 10~50。設太低可能影響同公司/學校多人共用同一 IP 的正常用戶。

12. 常見問題

Q：為什麼投票後沒有觸發遊戲？

請檢查「每投 N 票觸發遊戲」的設定。若設為 3，則需投滿 3 票才會觸發一次遊戲。

Q：獎品庫存歸零後會怎樣？

該獎品會自動從抽獎池排除，其權重會分配給其他有庫存的獎品。若所有獎品都歸零，則一律返回「感謝參與」。

Q：如何更改管理員密碼？

目前需手動更新資料庫中 AdminUser 表的 passwordHash 欄位。

Q：500 人同時進入能承受嗎？

瀏覽首頁：完全沒問題（讀取操作是併發的）。
同時投票+抽獎：分散在幾分鐘內可承受。若需要瞬間高併發（倒數搶投），建議將資料庫升級為 PostgreSQL。

Q：訪客追蹤如何避免灌水？

每個設備每天只記錄一次首頁訪問、一次作品點閱，不會因重新整理而重複計數。
